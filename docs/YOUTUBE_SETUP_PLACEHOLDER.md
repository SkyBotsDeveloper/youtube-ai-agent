# YouTube Setup Placeholder

Real YouTube upload is implemented in Phase 5 as a private, approval-gated workflow. Analytics remain future work.

Future setup will require:

1. A Google Cloud project.
2. YouTube Data API v3 enabled.
3. YouTube Analytics API enabled.
4. OAuth consent screen configured.
5. OAuth client credentials.
6. A refresh token generated by an explicit user authorization flow.

Expected environment variables:

- `YOUTUBE_CLIENT_ID`
- `YOUTUBE_CLIENT_SECRET`
- `YOUTUBE_REFRESH_TOKEN`
- `YOUTUBE_TOKEN_URI`
- `YOUTUBE_SCOPES`

Future upload policy:

- Upload as `private` by default.
- Require human approval before scheduled or public release.
- Run at most one original daily Short unless the operator changes the policy.
- Store every upload attempt and resulting YouTube video ID in the database.
